## Video Generation Task

### Interface Request

| Type | Description |
| --- | --- |
| Transmission Method | https |
| Request Address | <https://open.bigmodel.cn/api/paas/v4/videos/generations> |
| Calling Method | Asynchronous call, need to obtain results through the query interface |
| Character Encoding | UTF-8 |
| Interface Request Format | JSON |
| Response Format | JSON |
| Interface Request Type | POST |
| Development Language | Any development language that can initiate HTTP requests |

### Request Parameters

| Parameter Name | Type | Required | Parameter Description |
| --- | --- | --- | --- |
| model | String | Yes | Model code |
| prompt | String | Optional | Text description of the video, with a maximum input length of 512 Tokens. Either image\_url and prompt can be used, or both can be passed simultaneously. |
| quality | String | No | Output mode, default is `"speed"`. `"quality"`: quality priority, high generation quality. `"speed"`: speed priority, faster generation time, relatively lower quality. |
| with\_audio | Bool | No | Whether to generate AI sound effects. Default value: False (do not generate sound effects). |
| image\_url | String | Optional | Provide an image based on which content will be generated. If this parameter is passed, the system will operate based on this image. Supports passing images via URL or Base64 encoding. Image requirements: images support .png, .jpeg, .jpg formats; image size: no more than 5M. Either image\_url and prompt can be used, or both can be passed simultaneously.  First and last frames: supports inputting two images. The first uploaded image is regarded as the first frame, and the second image is regarded as the last frame. The model will generate the video based on the images passed in this parameter. First and last frame mode only supports `speed` mode |
| size | String | No | Default value: if not specified, the short side of the generated video is 1080 by default, and the long side is determined according to the original image ratio. Maximum support for 4K resolution. Resolution options: `"1280x720"`, `"720x1280"`, `"1024x1024"`, `"1920x1080"`, `"1080x1920"`, `"2048x1080"`, `"3840x2160"` |
| fps | int | No | Video frame rate (FPS), optional values are 30 or 60. Default value: 30. |
| duration | int | No | Video duration, default is 5 seconds, supports 5 and 10 seconds. |
| request\_id | String | No | Task number submitted by the user in the client request or generated by the platform; must be unique. If not passed by the client, the platform will generate it by default. |
| user\_id | String | No | Unique ID of the end user, helping the platform to intervene in the end user's violations, generation of illegal and harmful information, or other abusive behaviors. ID length requirement: at least 6 characters, at most 128 characters. [Learn more](#) |

### Response Parameters

| Parameter Name | Type | Parameter Description |
| --- | --- | --- |
| request\_id | String | Task number submitted by the user in the client request or generated by the platform |
| id | String | Task order number generated by Zhipu AI Open Platform, please use this order number when calling the result query interface |
| model | String | Name of the model called this time |
| task\_status | string | Processing status: PROCESSING (processing), SUCCESS (success), FAIL (failure). Results need to be obtained through query |

### Call Examples

#### Text to Video

```
from zhipuai import ZhipuAI
client = ZhipuAI(api_key="") # Please fill in your own APIKey

response = client.videos.generations(
    model="cogvideox-3",
    prompt="Peter Rabbit drives a car, wandering on the road, with a happy expression on his face.",
    quality="quality",  # Output mode, "quality" for quality priority, "speed" for speed priority
    with_audio=True,
    size="1920x1080",  # Video resolution, supports up to 4K (e.g.: "3840x2160")
    fps=30,  # Frame rate, optional 30 or 60
)
print(response)
```

## Task Result Query

### Interface Request

| Type | Description |
| --- | --- |
| Transmission Method | https |
| Request Address | https://open.bigmodel.cn/api/paas/v4/async-result/{id} |
| Calling Method | Synchronous call, wait for the model to complete execution and return the final result |
| Character Encoding | UTF-8 |
| Interface Request Format | JSON |
| Response Format | JSON |
| Interface Request Type | GET |
| Development Language | Any development language capable of initiating HTTP requests |

### Request Parameters

| Parameter Name | Type | Required | Parameter Description |
| --- | --- | --- | --- |
| id | String | Yes | Task ID |

### Response Parameters

| Parameter Name | Type | Parameter Description |
| --- | --- | --- |
| model | String | Model name |
| video\_result | List | Video generation results |
| url | String | Video URL |
| cover\_image\_url | String | Video cover image URL |
| task\_status | String | Processing status: PROCESSING (In progress), SUCCESS (Completed), FAIL (Failed). Note: For processing status, results need to be obtained via query. |
| request\_id | String | Task number submitted by the user in the client request or generated by the platform |

### Call Example

```
from zhipuai import ZhipuAI
client = ZhipuAI(api_key="") # Please fill in your own APIKey

response = client.videos.retrieve_videos_result(
    id="8868902201637896192"
)
print(response)
```
