# Tokenizer

Supported models: glm-4-plus, glm-4-long, glm-4-0520, glm-4-air, glm-4-flash

## **API Request**

| **Transfer Method** | **https** |
| --- | --- |
| **Request URL** | https://open.bigmodel.cn/api/paas/v4/tokenizer |
| **Calling Method** | Synchronous call, waiting for the system to complete execution and return the final result |
| **Character Encoding** | UTF-8 |
| **Request Format** | JSON |
| **Response Format** | JSON |
| **Request Type** | POST |
| **Development Language** | Any development language that can initiate HTTP requests |

## API Request Parameters

| **Parameter Name** | **Type** | **Required** | **Parameter Description** |
| --- | --- | --- | --- |
| model | String | Yes | The model code to be called. |
| messages | List<Object> | Yes | The current conversation message list as the prompt input for the language model, provided in JSON array format, e.g., {‚Äúrole‚Äù: ‚Äúuser‚Äù, ‚Äúcontent‚Äù: ‚ÄúHello‚Äù}. Possible message types include system messages, user messages, assistant messages, and tool messages. |
| request\_id | String | No | Passed by the client, needs to be unique; used to distinguish each request. If not provided by the client, the platform will generate one by default. |
| user\_id | String | No | The unique ID of the end user, helping the platform intervene in illegal activities, generate illegal or improper information, or other abuse by the end user. ID length requirements: at least 6 characters, up to 128 characters. |

#### API Response Parameters

| **Parameter Name** | **Type** | **Parameter Description** |
| --- | --- | --- |
| request\_id | String | The task ID submitted by the client when the request was initiated or the task ID generated by the platform. |
| id | String | The task sequence number generated by the Zhipu AI Open Platform, use this sequence number when calling the request result interface. |
| created | Long | The creation time of the request, in Unix timestamp format, in seconds. |
| usage | Object | The token statistics returned when the tokenizer calculation ends, does not actually consume tokens |
| prompt\_tokens | Integer | The number of tokens in the user input |

### HTTP Call Example

### **Request Example**

```
curl --location --request POST 'https://open.bigmodel.cn/api/paas/v4/tokenizer' \
--header 'Authorization: Your own APIKey' \
--header 'Content-Type: application/json' \
--data-raw '{
    "model":"glm-4-plus",
    "messages":[
        {
            "role":"user",
            "content":"Welcome to the Zhipu AI Open Platform"
        }
    ]
}'
```
