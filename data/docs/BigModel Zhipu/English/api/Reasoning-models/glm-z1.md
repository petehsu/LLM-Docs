## Synchronous Invocation

### Interface Information

| Type | Description |
| --- | --- |
| Method | HTTPS |
| Request URL | `https://open.bigmodel.cn/api/paas/v4/chat/completions` |
| Invocation Mode | Synchronous, waits for the model to complete execution and returns the final result or uses SSE invocation |
| Character Encoding | UTF-8 |
| Request Format | JSON |
| Response Format | JSON or standard Stream Event |
| Request Type | POST |
| Development Language | Any language capable of making HTTP requests |

### Request Parameters

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |
| model | String | Yes | The model code to be invoked. |
| messages | List<Object> | Yes | When invoking the language model, the current conversation message list serves as the model‚Äôs prompt input, provided in JSON array format, e.g., `{"role": "user", "content": "Hello"}`. Possible message types include system messages, user messages, assistant messages, and tool messages. |
| request\_id | String | No | Passed by the client; must be unique. Used as a unique identifier for each request. If not provided by the client, the platform will generate one by default. |
| do\_sample | Boolean | No | When `do_sample` is `true`, sampling strategies are enabled; when `false`, sampling strategy parameters like `temperature` and `top_p` will not take effect. Default is `true`. |
| stream | Boolean | No | This parameter should be set to `false` or omitted for synchronous invocation. Indicates that the model returns all content at once after generating everything. Default is `false`. If set to `true`, the model will return generated content in chunks via standard Event Stream. When the Event Stream ends, a `data: [DONE]` message will be returned. |
| temperature | Float | No | Sampling temperature, controls the randomness of the output. Must be a positive number in the range `[0.0, 1.0]`. Default is `0.95`. |
| top\_p | Float | No | An alternative method for temperature sampling, with a range of `[0.0, 1.0]`. Default is `0.7`. |
| max\_tokens | Integer | No | The maximum number of tokens for model output. The maximum output is 30k. |
| stop | List | No | The model will stop generating when encountering the specified stop characters. Currently, only a single stop word is supported, formatted as `["stop_word1"]`. |
| user\_id | String | No | The unique ID of the end-user, helping the platform intervene in cases of illegal activities, generating inappropriate content, or other abusive behaviors by end-users. ID length requirement: at least 6 characters, up to 128 characters. |

### Message Format

#### System Message

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |

| content | String | Yes | The message content. |

#### User Message

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |
| role | String | Yes | The role information of the message, which should be `user`. |

#### Assistant Message

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |
| role | String | Yes | The role information of the message, which should be `assistant`. |
| content | String | Yes | The message content. |

| Parameter Name | Type | Description |
| --- | --- | --- |
| id | String | Task ID. |
| created | Long | Request creation time, in Unix timestamp (seconds). |

| choices | List | The model‚Äôs output content for the current conversation. |

```
import time
from zhipuai import ZhipuAI

client = ZhipuAI(api_key="")  # Fill in your own APIKey

response = client.chat.asyncCompletions.create(
    model="glm-z1-airx",  # Fill in the model name you want to invoke
    messages=[
        {"role": "user", "content": "A bag contains 5 red balls and 3 blue balls. If 2 balls are drawn at random, the probability of drawing at least 1 red ball is:"}
    ],
)
task_id = response.id
task_status = ''
get_cnt = 0

while task_status != 'SUCCESS' and task_status != 'FAILED' and get_cnt <= 40:
    result_response = client.chat.asyncCompletions.retrieve_completion_result(id=task_id)
    print(result_response)
    task_status = result_response.task_status

    time.sleep(2)
    get_cnt += 1
```
