# Снятие моделей с поддержки

Информация о снятии моделей Claude с поддержки, сроках выхода на пенсию и рекомендуемых заменах.

---

По мере запуска более безопасных и мощных моделей мы регулярно снимаем с поддержки старые модели. Приложения, использующие модели Anthropic, могут потребовать периодических обновлений для продолжения работы. Затронутые клиенты всегда будут уведомлены по электронной почте и в нашей документации.

На этой странице перечислены все устаревания API вместе с рекомендуемыми заменами.

## Обзор

Anthropic использует следующие термины для описания жизненного цикла наших моделей:
- **Active**: Модель полностью поддерживается и рекомендуется для использования.
- **Legacy**: Модель больше не будет получать обновления и может быть снята с поддержки в будущем.
- **Deprecated**: Модель больше не доступна для новых клиентов, но продолжает быть доступной для существующих пользователей до снятия с поддержки. На этом этапе мы назначаем дату снятия с поддержки.
- **Retired**: Модель больше не доступна для использования. Запросы к снятым с поддержки моделям будут отклонены.

<Warning>
Обратите внимание, что снятые с поддержки модели, вероятно, будут менее надежными, чем активные модели. Мы настоятельно рекомендуем вам перенести рабочие нагрузки на активные модели, чтобы поддерживать наивысший уровень поддержки и надежности.
</Warning>

## Миграция на замены

После того как модель снята с поддержки, пожалуйста, перенесите все использование на подходящую замену до даты снятия с поддержки. Запросы к моделям после даты снятия с поддержки будут отклонены.

Чтобы помочь измерить производительность моделей-замен на ваших задачах, мы рекомендуем тщательное тестирование ваших приложений с новыми моделями задолго до даты снятия с поддержки.

Для получения конкретных инструкций по миграции с Claude 3.7 на модели Claude 4.5 см. [Миграция на Claude 4.5](/docs/ru/about-claude/models/migrating-to-claude-4).

## Уведомления

Anthropic уведомляет клиентов с активными развертываниями о моделях с предстоящим снятием с поддержки. Мы предоставляем как минимум 60 дней уведомления перед снятием модели с поддержки для публично выпущенных моделей.

## Аудит использования моделей

Чтобы помочь определить использование снятых с поддержки моделей, клиенты могут получить доступ к аудиту использования их API. Выполните следующие шаги:

1. Перейдите на страницу [Usage](/settings/usage) в Console
2. Нажмите кнопку "Export"
3. Просмотрите загруженный CSV, чтобы увидеть использование, разбитое по ключам API и моделям

Этот аудит поможет вам найти любые случаи, когда ваше приложение все еще использует снятые с поддержки модели, позволяя вам приоритизировать обновления на более новые модели до даты снятия с поддержки.

## Лучшие практики

1. Регулярно проверяйте нашу документацию на предмет обновлений о снятии моделей с поддержки.
2. Тестируйте ваши приложения с более новыми моделями задолго до даты снятия вашей текущей модели с поддержки.
3. Обновите ваш код, чтобы использовать рекомендуемую модель-замену как можно скорее.
4. Свяжитесь с нашей командой поддержки, если вам нужна помощь с миграцией или у вас есть какие-либо вопросы.

## Недостатки снятия с поддержки и их смягчение

Мы в настоящее время снимаем с поддержки и выводим модели из обращения, чтобы обеспечить емкость для новых выпусков моделей. Мы признаем, что это имеет недостатки:
- Пользователи, которые ценят конкретные модели, должны перейти на новые версии
- Исследователи теряют доступ к моделям для текущих и сравнительных исследований
- Снятие модели с поддержки вводит риски, связанные с безопасностью и благополучием модели

В какой-то момент мы надеемся снова сделать прошлые модели общедоступными. Тем временем мы взяли на себя обязательство по долгосрочному сохранению весов моделей и другим мерам, чтобы помочь смягчить эти последствия. Для получения дополнительной информации см. [Commitments on Model Deprecation and Preservation](https://www.anthropic.com/research/deprecation-commitments).
