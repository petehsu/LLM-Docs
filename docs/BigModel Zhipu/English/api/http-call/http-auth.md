# HTTP User Authentication

**When calling the model interface, two authentication methods are supported**:

* Pass the API Key for authentication
* Pass the authentication token for authentication

The current platform authentication token is generated by the user side, using the creation method provided by the standard JWT (for details, refer to: https://jwt.io/introduction).

## Obtain Your API Key

Log in to the Zhipu AI Open Platform [API Keys page](https://bigmodel.cn/usercenter/apikeys)  to get the latest generated user API Key.

In the new mechanism, the platform-issued API Key contains both the ‚Äúuser identifier id‚Äù and the ‚Äúsignature secret‚Äù, i.e., the format is `{id}.{secret}`.

## Use API Key for Request

Users need to place the API Key in the Authorization header of the HTTP request.

Example: API key parameter example in curl request

```
curl --location 'https://open.bigmodel.cn/api/paas/v4/chat/completions' \
--header 'Authorization: Bearer <your_apikey>' \
--header 'Content-Type: application/json' \
--data '{
    "model": "glm-4-air-250414",
    "messages": [
```

## Use JWT to Assemble Token for Request

The user side needs to introduce the corresponding JWT-related tools and assemble the header and payload parts of the JWT as follows:

#### 1. Header example

```
{"alg":"HS256","sign_type":"SIGN"}
```

1

* `alg`: Attribute indicates the algorithm used for signing, defaulting to HMAC SHA256 (written as HS256).
* `sign_type`: Attribute indicates the type of the token, with JWT tokens uniformly written as SIGN.

#### 2. Payload example

```
{"api_key":{ApiKey.id},"exp":1682503829130, "timestamp":1682503820130}
```

1

* `api_key`: Attribute indicates the user identifier id, i.e., the `{id}` part of the user API Key.
* `exp`: Attribute indicates the expiration time of the generated JWT, controlled by the client, in milliseconds.
* `timestamp`: Attribute indicates the current timestamp, in milliseconds.

Example: Authentication token assembly process in Python

```
import time
import jwt
 
def generate_token(apikey: str, exp_seconds: int):
    try:
        id, secret = apikey.split(".")
    except Exception as e:
        raise Exception("invalid apikey", e)
 
    payload = {
        "api_key": id,
        "exp": int(round(time.time() * 1000)) + exp_seconds * 1000,
        "timestamp": int(round(time.time() * 1000)),
    }
 
    return jwt.encode(
        payload,
        secret,
        algorithm="HS256",
        headers={"alg": "HS256", "sign_type": "SIGN"},
    )
```
