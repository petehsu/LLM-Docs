## Video Generation Task

### Interface Request

| Type | Description |
| --- | --- |
| Transmission | https |
| Request URL | https://open.bigmodel.cn/api/paas/v4/videos/generations |
| Call Method | Asynchronous call, results need to be retrieved via a query interface |
| Character Encoding | UTF-8 |
| Request Format | JSON |
| Response Format | JSON |
| Request Type | POST |
| Development Language | Any language capable of making HTTP requests |

### Request Parameters

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |
| model | String | Yes | Model code |
| prompt | String | Optional | Text description of the video, maximum input length of 512 Tokens. Either image\_url or prompt must be provided, or both. |
| quality | String | No | Output mode, default is `"speed"`. `"quality"`: Prioritizes quality, higher generation quality. `"speed"`: Prioritizes speed, faster generation time, relatively lower quality. |
| with\_audio | Bool | No | Whether to generate AI sound effects. Default: `False` (no sound effects). |
| image\_url | String | Optional | Image to base the content generation on. If provided, the system will operate based on this image. Supports URL or Base64 encoded images. Image requirements: Supports .png, .jpeg, .jpg formats; image size: no more than 5M. Either image\_url or prompt must be provided, or both. |
| size | String | No | Default: If not specified, the short side of the generated video defaults to 1080, with the long side scaled according to the original image ratio. Supports up to 4K resolution. Resolution options: 720x480, 1024x1024, 1280x960, 960x1280, 1920x1080, 1080x1920, 2048x1080, 3840x2160 |
| fps | int | No | Video frame rate (FPS), optional values are `30` or `60`. Default: `30`. |
| request\_id | String | No | Provided by the client, must be unique; used to distinguish each request‚Äôs unique identifier. If not provided by the client, the platform will generate one by default. |
| user\_id | String | No | Unique ID of the end-user, assists the platform in intervening in end-user violations, generating illegal or inappropriate information, or other abusive behaviors. ID length requirement: minimum 6 characters, maximum 128 characters. [Learn more](https://bigmodel.cn/dev/howuse/securityaudit) |

Note: cogvideox-flash does not support quality, size, fps parameter settings.

### Response Parameters

| Parameter Name | Type | Description |
| --- | --- | --- |
| request\_id | String | Task number submitted by the user during the client request or generated by the platform |
| id | String | Task order number generated by the ZhipuAI open platform, use this order number when calling the request result interface |
| model | String | Model name used in this call |
| task\_status | string | Processing status, PROCESSING (processing), SUCCESS (success), FAIL (failure). Results need to be obtained via query |

### Call Example

**Text to Video**

```
from zhipuai import ZhipuAI
client = ZhipuAI(api_key="") # Please fill in your own APIKey

response = client.videos.generations(
    model="cogvideox-2",
    prompt="Peter Rabbit drives a car, wandering on the road, with a happy and joyful expression on his face.",
    quality="quality",  # Output mode, "quality" for quality priority, "speed" for speed priority
    with_audio=True,
    size="1920x1080",  # Video resolution, supports up to 4K (e.g., "3840x2160")
    fps=30,  # Frame rate, optional 30 or 60
)
print(response)
```

## Task Result Query

### Interface Request

| Transmission | https |
| --- | --- |
| Request URL | https://open.bigmodel.cn/api/paas/v4/async-result/{id} |
| Call Method | Synchronous call, waits for the model to complete and returns the final result |
| Character Encoding | UTF-8 |
| Request Format | JSON |
| Response Format | JSON |
| Request Type | GET |
| Development Language | Any language capable of making HTTP requests |

### Request Parameters

| Parameter Name | Type | Required | Description |
| --- | --- | --- | --- |
| id | String | Yes | Task id |

### Response Parameters

| Parameter Name | Type | Description |
| --- | --- | --- |
| model | String | Model name |
| video\_result | List | Video generation result |
| url | String | Video url |
| cover\_image\_url | String | Video cover url |
| task\_status | String | Processing status, PROCESSING (processing), SUCCESS (success), FAIL (failure) Note: Processing status needs to be obtained via query |
| request\_id | String | Task number submitted by the user during the client request or generated by the platform |

### Call Example

#### Request Example

```
from zhipuai import ZhipuAI
client = ZhipuAI(api_key="") # Please fill in your own APIKey

response = client.videos.retrieve_videos_result(
    id="8868902201637896192"
)
print(response)
```
