# Content Security

Content in **text, image, audio, and video** formats can be detected to accurately identify risks such as pornography, violence, illegal content, and other violations. It provides structured moderation results (including content type, risk type, and specific risky content segments) for quick identification and handling of non-compliant information.

## API Call

| Type | Description |
| --- | --- |
| Method | HTTPS |
| Request URL | `https://open.bigmodel.cn/api/paas/v4/moderations` |
| Call Mode | Synchronous call, waits for the model to complete execution and returns the final result. |
| Encoding | UTF-8 |
| Request Format | JSON |
| Response Format | JSON |
| Request Type | POST |
| Development Language | Any language capable of making HTTP requests |

### **Request Parameters**

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `input` | String / Object | Yes | Content to be moderated:  ‚Ä¢ Plain text string (max length: 2000 characters)  ‚Ä¢ Multimedia object (`type` + `url`):   - Image: <10MB, min resolution 20x20, max resolution 6000x6000   - Video: Recommended duration ‚â§30s   - Audio: Recommended duration ‚â§60s |

### **Response Parameters**

| Parameter | Type | Description |
| --- | --- | --- |
| `request_id` | String | Request tracking ID |
| `processed_time` | Int | Server-side processing time (ms) |
| `results` | Array | Moderation results: |
| `content_type` | String | Content type (e.g., text, image, video, audio) |
| `risk_level` | String | Risk assessment result:  `PASS`: Normal content  `REVIEW`: Suspicious content  `REJECT`: Violative content |
| `risk_type` | Array | List of risk types |
| `risk_content` | String | Original risky content (for text-based moderation) |
